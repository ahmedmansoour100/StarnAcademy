أنشئ مشروع ويب متكامل وذو جودة إنتاجية عالية لموقع أكاديمية تعليمية باسم Starn Academy بحيث ينتج النموذج: كود قابل للتشغيل، ملفات تصميمية، وملف مضغوط ZIP يحتوي على المشروع كاملاً. المتطلبات التفصيلية أدناه يجب اتباعها حرفياً.

مخرجات متوقعة من النموذج

مشروع React كامل مع Tailwind CSS (أو بديل واضح قابل للإعداد) في مجلد واحد، أو بديل HTML/CSS/JS كامل ومرتب؛ كل ملف مع تعليقات توضيحية.

ملف README.md يشرح خطوات التشغيل محليًا (npm install, npm run dev/build) وأين تُضع مفاتيح الـ API إن وجدت.

مجلد assets يحتوي على: favicon، logo (مقاسات متعددة)، UI Kit مبسط (SVG/PNG)، وصور افتراضية للمشاريع.

ملف بيانات تجريبي (JSON) يحتوي على: 12 مشروع طلابي كما في المواصفات، و40 تقييمًا عشوائيًا بالقيم بين 31 و40 من 40.

صفحة واجهة متكاملة متجاوبة تعمل على Desktop وMobile مع سلوكين مختلفين للـ navigation: نسخة ويب بشريط علوي Top Bar، ونسخة موبايل بقائمة جانبية Side Drawer.

نموذج تسجيل/تسجيل دخول مع حفظ مؤقت للبيانات في localStorage (أو ملف JSON) وواجهة لربط حجوزات الأكاديمية بجوجل شيت (مثال تنفيذ عبر Google Sheets API أو تعليمات Google Apps Script مع كود نموذجي).

مخرجات ملفات محددة (تنسيق)

/README.md — أوامر التشغيل، وصف المشروع، كيف تغير ألوان الهوية من صورة اللون.

/package.json — إن وُجد React.

/src/components/* — مكونات قابلة لإعادة الاستخدام مع تعليقات.

/public/assets/* — الصور واللوجو والفافيكون.

/data/sample-data.json — بيانات الطلاب، المشاريع، 40 تقييمًا.

/scripts/google-apps-script.gs — (اختياري) مثال Google Apps Script لاستقبال حجوزات.

/build or /dist — نسخة جاهزة للنشر (إن أمكن توليد ملفات مُصنّعة).
